# ADR-0003 `/team-beta/` と `/estimate-audit/` のページ分離

## ステータス: Accepted (2026-07-10)

## 背景
- 1 ページに受託チーム・フリーランス・発注企業を同強度で載せるとメッセージが混ざる（監査 §2, §9）。
- 主対象は 2〜10 人の受託開発チーム。発注企業は別導線。

## 決定
1. **ホーム**は受託チーム向け Team Beta を主軸にする。
2. **`/team-beta/`**（または同等）: 月額・3か月検証・支援範囲・成果物・導入初月の流れ。主 CTA = Team Beta 相談。
3. **`/estimate-audit/`**: 発注企業向け。監査レポートの価値・対象・価格帯の目安。主 CTA = Estimate Audit 相談。
4. Header 推奨:
   - できること
   - 対象チーム
   - 料金
   - Estimate Audit
   - FAQ
   - Team Betaに相談
5. 背景動画は不要。白背景 + 実画面（またはプレースホルダ）中心。

## 理由
- 対象の分離で CTA の文脈と intent が明確になる。
- ページ長の問題を分割で緩和する。

## 影響
- 新規 pages + nav + meta / JSON-LD。
- ホームから両ページへの内部リンク。

## 代替案
- **単一 LP のままアンカー分割のみ**: 短期可だが監査の「2ページへ分離」を満たさないため中期は本 ADR を採る。
